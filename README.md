# Flight Booking Service

A small imaginary flight booking service.

The project explores concepts such as:

- Clean Architecture
- Hexagonal Architecture
- Domain-Driven Design (DDD)
- Test-Driven Development (TDD)

The goal isn't to build a production-ready application, but to better understand how these ideas work together.

## Why TDD?

Over time, I realized that good tests not only catch bugs but also help explain how the system is supposed to work.

## Purpose

This repository is just a place to experiment, learn, and try different approaches.
